O que são módulos
	
	Arquivos Javascript que tem capacidade de exportar e importar informações de outros arquivos do mesmo tipo.

	Algumas vantagens são:

- Organização do código;
- Compartilhamento de variáveis em escopos diferentes
- Explicita as dependências dos arquivos

	Exportar

Named exports: Quando deseja exportar mais de um arquivo e exporta cada um individualmente. Pode ser feita utilizando a palavra export antes da função ou ao final utilizando a palavra export e entre chaves os nomes das funções que deseja exportar.

Default exports: Só pode ter um default export por arquivo e será o retorno padrão do seu arquivo.

	Importar

Named exports: import {função, variável, classe} from ‘./arquivo.js’

Default exports: import valor default from ‘./arquivo.js’

Importando todos os dados de um arquivo: import * as INFOS from ‘./arquivo.js’

	Trocando nome de imports

Import { arquivo as Apelido } from ‘./arquivo.js’

Vinculando ao HTML
<script type=”module” src=”./main.js></script>

Para fazer testes localmente (de um arquivo no seu computador), será necessário estar rodando um servidor. Isso pode ser feito utilizando a extensão “Live Server”, do VScode.

	Outras curiosidades sobre módulo

- Módulos sempre estão em “strict mode”; ( Javascript com um pouco mais de regras)
- Podem ser utilizadas as extensões .js e .mjs
- Para testes locais, é necessário utilizar um servidor
- Ao importar, sempre lembra da extensão
- Ao importar, sempre utiliza “./” como ponto de partida.
